This guide walks you through creating and using Balid schemas for runtime validation.

## Getting started

<Steps>
  <Step title="Import Balid">
    Import the `b` object from Balid, which provides access to all validators:

    ```typescript
    import { b } from "balid";
    ```
  </Step>

  <Step title="Create a schema">
    Use validator methods to define your data structure. Let's start with a simple string validator:

    ```typescript
    const schema = b.string();
    ```
  </Step>

  <Step title="Validate data">
    Call the `validate` method with your input data:

    ```typescript
    const result = schema.validate("Hello World");
    console.log(result); // { valid: true, errors: [] }
    ```
  </Step>
</Steps>

## Validation response structure

Every validation returns a `Valid` object with two properties:

```typescript
type Valid = {
  valid: boolean;   // true if validation passed, false otherwise
  errors: string[]; // array of error messages (empty if valid)
};
```

**Successful validation:**
```typescript
const schema = b.boolean();
schema.validate(false); // { valid: true, errors: [] }
```

**Failed validation:**
```typescript
const schema = b.boolean();
schema.validate("false"); // { valid: false, errors: ["Not a boolean"] }
```

## Basic validators

### String validation

Validate strings with optional constraints:

```typescript
import { b } from "balid";

// Basic string validation
const schema = b.string();
schema.validate("Hello World"); // { valid: true, errors: [] }

// String with length constraints
const minMaxSchema = b.string({ min: 2, max: 10 });
minMaxSchema.validate("a"); // { valid: false, errors: [...] }
minMaxSchema.validate("hello"); // { valid: true, errors: [] }

// String with pattern matching
const matchSchema = b.string({ match: /hello/ });
matchSchema.validate("hello"); // { valid: true, errors: [] }
matchSchema.validate("world"); // { valid: false, errors: [...] }
```

### Number validation

Validate numbers with range constraints:

```typescript
import { b } from "balid";

// Basic number validation
const schema = b.number();
schema.validate(123); // { valid: true, errors: [] }
schema.validate("123"); // { valid: false, errors: ["Not a number"] }

// Number with min/max constraints
const minMaxSchema = b.number({ min: 10, max: 20 });
minMaxSchema.validate(1); // { valid: false, errors: [...] }
minMaxSchema.validate(25); // { valid: false, errors: [...] }
minMaxSchema.validate(15); // { valid: true, errors: [] }
```

### Object validation

Validate complex objects with nested schemas:

```typescript
import { b } from "balid";

const schema = b.object({
  name: b.string(),
});

schema.validate({ name: "John Doe" }); // { valid: true, errors: [] }
```

<Tip>
  Object validation returns detailed error messages with property paths when nested validations fail, like `"name: Not a string"`.
</Tip>

### Array validation

Validate arrays where all elements match a schema:

```typescript
import { b } from "balid";

const schema = b.array(b.string());

schema.validate(["John Doe"]); // { valid: true, errors: [] }
```

## Combining validators

Balid provides utility functions to compose schemas:

```typescript
import { b } from "balid";

// Optional values (undefined or matches schema)
const optionalSchema = b.optional(b.string());
optionalSchema.validate("hello world"); // { valid: true, errors: [] }
optionalSchema.validate(undefined); // { valid: true, errors: [] }

// Or logic (matches any of the schemas)
const orSchema = b.or({}, b.string(), b.boolean());
orSchema.validate("hello world"); // { valid: true, errors: [] }
orSchema.validate(true); // { valid: true, errors: [] }

// And logic (matches all schemas)
const andSchema = b.and({}, b.any(), b.boolean());
andSchema.validate(true); // { valid: true, errors: [] }
```
